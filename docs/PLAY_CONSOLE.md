# Play Store: local build → closed test

How to ship Cryptogram to a Google Play **closed test** with a **locally built**
AAB (no EAS). Companion to [RELEASE.md](./RELEASE.md) and [PUBLISHING.md](./PUBLISHING.md).

> Note: `android/` is gitignored (Expo regenerates it on prebuild). The release
> signing wiring in `android/app/build.gradle` is therefore **not committed** and
> is lost on `expo prebuild --clean`. To make it permanent, move it into a config
> plugin (see "Persisting the signing config" at the bottom).

---

## 1. Create the upload keystore (one time)

From `android/app/`:

```
keytool -genkeypair -v -keystore upload-keystore.jks -alias upload \
  -keyalg RSA -keysize 2048 -validity 10000
```

- It prompts for a **keystore password**, a **key password**, and a name/org (any
  values — CN can be "Shareef Solutions").
- **Back this file + passwords up safely.** It's your upload identity for every
  future update. (With Play App Signing you *can* reset a lost upload key via
  Google support, but don't rely on it.)
- The `.jks` is gitignored. If `keytool` isn't on PATH, it's in Android Studio's
  JBR: `C:\Program Files\Android\Android Studio\jbr\bin\keytool.exe`.

## 2. Point Gradle at the key (one time)

Put credentials in your **global** Gradle properties (never committed):
`C:\Users\admin\.gradle\gradle.properties`

```
MYAPP_UPLOAD_STORE_FILE=upload-keystore.jks
MYAPP_UPLOAD_KEY_ALIAS=upload
MYAPP_UPLOAD_STORE_PASSWORD=<keystore password>
MYAPP_UPLOAD_KEY_PASSWORD=<key password>
```

`android/app/build.gradle` already reads these (release signingConfig); when set,
the `release` build type is signed with your upload key instead of debug.

## 3. Build the AAB

```
cd android
./gradlew bundleRelease          # Windows: .\gradlew.bat bundleRelease
```

Output: `android/app/build/outputs/bundle/release/app-release.aab`

Verify it's signed with the upload key (not debug):

```
keytool -printcert -jarfile app/build/outputs/bundle/release/app-release.aab
```

The owner/issuer should be your name/org — **not** `CN=Android Debug`.

> Bumping versions: each Play upload needs a higher `versionCode`. Edit
> `android/app/build.gradle` → `versionCode` (currently `1`) and `versionName`
> for each new upload.

---

## 4. Play Console — create & configure the app

In [play.google.com/console](https://play.google.com/console):

1. **Create app** → name "Cryptogram", Game, Free, accept declarations.
2. **Store listing** (Grow → Store presence): short + full description, app icon
   (512×512), **feature graphic 1024×500**, and **2–8 phone screenshots**.
3. **Privacy policy** (Policy → App content): `https://shareefsolutions.in/cryptogram/privacy`
4. **App access**: all functionality available without login (it is).
5. **Ads**: declare **Yes, contains ads**.
6. **Content rating**: complete the questionnaire (puzzle/no objectionable content).
7. **Target audience**: 13+ (not child-directed).
8. **Data safety**: declare what AdMob collects — Device or other IDs +
   App activity; used for Advertising/Marketing; not for tracking you control;
   match the privacy page.
9. **Government apps / News / COVID**: No.

## 5. In-app product (Remove Ads)

Monetize → Products → **In-app products** → Create:
- Product ID **`remove_ads`** (must match the app), set a price, Activate.
- IAP only transacts for testers on a track (next step), not local debug.

## 6. Closed testing track

Test and release → Testing → **Closed testing** → Create track:
1. Upload `app-release.aab`. (First upload enrolls **Play App Signing** — accept;
   Google holds the app signing key, your keystore is the upload key.)
2. Add a **testers** list — up to ~20 emails (Google accounts). For a personal
   developer account, the **14-day continuous closed-test** requirement runs from
   here.
3. Roll out to the track and share the **opt-in link** with testers.

## 7. Production

After the 14-day closed test + review: promote the build to **Production**,
complete any remaining declarations, and submit for review.

---

## Persisting the signing config (recommended)

Because `android/` is regenerated by prebuild, the cleanest long-term fix is a
local config plugin that injects the release signingConfig during
`expo prebuild`, so it survives regeneration and is tracked in git. Ask and it
can be added (a `plugins/withReleaseSigning.js` referenced from `app.json`).
